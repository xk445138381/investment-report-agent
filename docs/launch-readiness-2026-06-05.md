## Recommended Next Step

Prepare a deployment target and run the container build in an environment with Docker:

1. Build with `docker compose -f docker-compose.prod.yml build`.
2. Start with production env vars for `FRONTEND_ORIGINS`, `NEXT_PUBLIC_API_URL`, `DEEPSEEK_API_KEY`, and `QVERIS_API_KEY`.
3. Run `python scripts/predeploy_check.py --production-target ...` against the deployed target.
4. Verify `/health` returns 200 and `/ready` returns `status=ok` in the target environment.
5. Attach managed MongoDB or verify the compose MongoDB volume strategy.
6. Connect `/metrics`, logs, uptime checks, alerting, and rollback procedure before public traffic.
