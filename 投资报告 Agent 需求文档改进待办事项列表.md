# 投资报告 Agent 需求文档改进待办事项列表

本列表汇总了对《投资报告 Agent — 完整产品与架构方案》文档的审查建议，按模块分类，方便后续跟进完善。

## 1. 产品决策与竞品分析
- [ ] **明确决策优先级与依赖关系**：细化 15 项核心决策在不同开发阶段的执行顺序及相互依赖。
- [ ] **补充决策风险与缓解策略**：在每个决策点后增加潜在风险说明及应对方案。
- [ ] **扩展商业竞品分析**：补充 Bloomberg、Refinitiv 等商业终端报告功能的概览分析。
- [ ] **增加项目 SWOT 分析**：基于竞品分析，明确本项目的优势、劣势、机遇与威胁。

## 2. 数据层与 API
- [ ] **编制详细数据字典**：列出所有数据字段的名称、类型、单位、来源及更新频率。
- [ ] **细化数据质量保障措施**：补充数据校验规则、异常处理流程及数据回溯机制。
- [ ] **明确数据更新与同步策略**：定义不同数据源（行情、财报等）的具体同步频率。
- [ ] **集中管理数据模型**：将所有 Pydantic Schema 集中定义，并补充继承关系说明。
- [ ] **定义 RESTful API 接口规范**：使用 OpenAPI 规范详细描述各接口的请求与响应结构。

## 3. 系统架构与 Agent 设计
- [ ] **绘制 Agent 编排流程图**：可视化展示 Orchestrator 调度逻辑及 Agent 间的数据流。
- [ ] **完善错误处理与监控方案**：增加全局错误处理、重试机制、告警及性能监控设计。
- [ ] **强化系统安全性设计**：补充 API 认证授权、传输加密及容器安全等方案。
- [ ] **明确 Agent 间通信协议**：统一定义 Agent 间传递的数据格式，确保兼容性。
- [ ] **补充 LLM 选型详细理由**：阐述 Haiku/Sonnet/Opus 在不同场景下的适用性依据。

## 4. 计算层与模板系统
- [ ] **规划计算模块化设计**：将技术指标、财务比率等计算逻辑封装为独立模块。
- [ ] **实现计算参数配置化**：将 WACC、增长假设等关键参数从代码中抽离至配置文件。
- [ ] **探索计算性能优化**：针对计算密集型任务，评估 NumPy 向量化或并行计算方案。
- [ ] **设计模板管理界面**：规划用户上传、编辑、预览及版本管理自定义模板的 UI。
- [ ] **详述模板变量绑定规则**：说明结构化数据如何映射到 HTML 模板占位符。
- [ ] **明确图表嵌入技术方案**：确定图表在报告中的渲染与嵌入方式（Base64 或 链接）。

## 5. UI/UX 与质量保证
- [ ] **提供关键界面线框图/原型**：绘制核心交互界面的低保真原型。
- [ ] **融入用户反馈交互机制**：在 UI 中设计报告评分、修改建议等反馈入口。
- [ ] **规划多语言 UI 支持方案**：为国际化市场预留界面适配方案。
- [ ] **细化测试用例管理策略**：设计针对 Agent 复杂行为的测试场景与预期输出。
- [ ] **集成 CI/CD 自动化测试**：将测试策略融入自动化部署流程。
- [ ] **定义 MVP 成功指标与验收标准**：为 Phase 1 交付物设定可量化的考核指标。

## 6. Prompt 工程与合规
- [ ] **提供 Prompt 示例与模板**：为附录中的 Prompt 启示提供具体的应用示例。
- [ ] **建立 Prompt 版本管理机制**：将 Prompt 纳入版本控制系统进行迭代管理。
- [ ] **寻求专业法律咨询**：在启动前确认数据使用与 ToS 条款的合规性。
- [ ] **完善隐私政策说明**：明确用户上传数据的处理边界与隐私保护措施。
